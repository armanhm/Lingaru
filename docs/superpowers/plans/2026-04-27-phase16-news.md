# Phase 16 — News (dedicated practice surface)

**Date:** 2026-04-27
**Status:** Shipped
**App:** `apps.discover` (alternate URL namespace `/api/news/`)

## Why

The Discover feed historically mixed word, grammar, trivia, **and** news cards. News deserved more — a real article body, vocab/expression/grammar drill panes, audio. Cramming that into Discover's inline-expand pattern would either bloat Discover or short-change the news experience. So News got promoted to its own page, and Discover stopped showing news cards.

## Scope

- Standalone `/news` listing page (filterable by topic) and `/news/:id` detail page
- 9 topics: politics, sports, culture, economy, science, tech, society, environ, world
- Each article carries `vocabulary[]`, `expressions[]`, `grammar_points[]`
- Curated 9-topic offline mock library so the feature works without a live LLM
- News cards live for 7 days (vs 24h for Discover types)

## Backend

Reuses `DiscoverCard` with two new optional fields (migration `0002_discovercard_level_discovercard_topic`):

- `topic` — choices: `politics, sports, culture, economy, science, tech, society, environ, world, misc`
- `level` — CEFR `A1 | A2 | B1 | B2 | C1 | C2`

`content_json` shape for `type="news"`:

```json
{
  "article_fr": "...",
  "article_en": "...",
  "vocabulary":     [{ "french", "english", "pos", "example_fr" }],
  "expressions":    [{ "fr", "en", "note" }],
  "grammar_points": [{ "title", "explanation", "example_fr" }]
}
```

`apps.discover.services.generate_news_card(topic=None)` calls `services.llm` with the `news_generator` system prompt; on LLM failure it falls back to a curated library covering all 9 topics. `seed_news` management command bulk-loads them.

`apps.discover.views.FeedView` excludes `type="news"`. `generate_daily_cards()` no longer produces news.

## Endpoints (`/api/news/`)

Wired via `apps.discover.news_urls` mounted at `/api/news/` in `config/urls.py`.

| Method | Path | Purpose |
|---|---|---|
| GET | `` | Paginated list, optional `?topic=` filter, returns `{ articles, topics }` (topic counts) |
| GET | `{id}/` | Full article + vocab/expressions/grammar arrays |
| POST | `generate/` | Generate one (optional `{ "topic": "..." }`) |
| POST | `{id}/interact/` | Mark as read, award XP |

## Frontend

- `News` (listing) — PageHeader, 10 topic-filter pills (gradient when active), gradient top-band cards with topic badge, level chip (A1–C2 toned), read-time estimate, French timeAgo, "✓ Lu" indicator
- `NewsArticle` (detail) — full-width article body on top (FR with AudioPlayButton, summary intro, translation toggle to EN), full-width Pratique guidée section below with three large equal-width tabs (📚 Vocabulaire / 💬 Expressions / 🧠 Grammaire), each tab uses a multi-column grid for readability. Auto-marks as read (awards XP) on first view.
- Inter typography for short FR labels in vocab/expression rows (Instrument Serif italic was hard to read at small sizes)
- Sidebar nav entry placed right after Discover
