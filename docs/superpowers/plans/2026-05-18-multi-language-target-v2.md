# Multi-Language Target -- v2.0.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the v2.0.0 multi-language refactor in a single PR. Adds `target_language` to User, a `language` discriminator field to 13 content tables, per-language LLM prompts, per-agent EN system prompt fallback, an onboarding language step, a Settings target_language picker, a flag indicator next to the logo, "coming soon" badges for EN-unsupported features, and an English content seed (5 topics, 25 lessons, ~300 vocab, ~10 news articles).

**Architecture:** Approach A from the spec -- a `language` CharField added to every content table with `default="fr"`, all queries filter by `request.user.target_language`. No new abstractions, just a discriminator field. Per-language LLM prompts live in a 2-level dict `{language -> {mode -> prompt}}` accessed via a helper. The new shared `LANGUAGE_CHOICES` constant lives in `apps/users/constants.py` to break import cycles.

**Tech Stack:** Django 5.1 + DRF + pytest (backend), React 18 + Vite + Tailwind + react-i18next (frontend), PostgreSQL prod / SQLite for tests.

**Spec:** [`docs/superpowers/specs/2026-05-18-multi-language-target-design.md`](../specs/2026-05-18-multi-language-target-design.md)

## Conventions to follow

- **Backend tests:** pytest, `config.settings.test`, fixtures + `@pytest.mark.django_db`. SQLite.
- **Backend code style:** Ruff lint + format. `ruff format <paths> && ruff check <paths>` from `backend/` before each commit.
- **Frontend code style:** ESLint + Vite build. `npm run lint && npm run build` from `frontend/`.
- **Commits:** Conventional Commits (`feat:`, `refactor:`, `test:`, `docs:`).
- **No em-dashes** in user-facing strings.
- **404 vs 200**: detail/slug cross-language access returns 404; list endpoints return 200 with empty data.
- **No new env vars.** v2.0.0 doesn't add feature flags.

import { useTranslation } from "react-i18next";

const FLAG_FOR = { fr: "🇫🇷", en: "🇬🇧" };

/**
 * Read-only flag emoji indicating the user's current target_language.
 * Sits next to the logo in the layout header on every authed page.
 * Hovering shows a tooltip explaining how to change it (Settings).
 *
 * Returns null for unknown languages so a future migration that
 * introduces a 3rd language but forgets to update FLAG_FOR doesn't
 * crash the layout -- it just renders nothing.
 */
export function LearningLanguageFlag({ language }) {
  const { t } = useTranslation();
  const NAME_FOR = { fr: t("languages.fr"), en: t("languages.en") };
  const flag = FLAG_FOR[language];
  if (!flag) return null;
  return (
    <span
      role="img"
      aria-label={t("layout.learningLanguageAria", { language: NAME_FOR[language] })}
      title={t("layout.learningLanguageTooltip", { language: NAME_FOR[language] })}
      className="ml-1 text-lg leading-none select-none"
    >
      {flag}
    </span>
  );
}

                    question_text=mcq_text,
                    defaults={
                        "question_type": "mcq",
                        "correct_answer": vocab.french,
                        "options": [vocab.french] + distractors,
                        "language": "en",
                    },
                )
                if created:

                # Fill-blank: "<English sentence with one word blanked>"
